Model Evaluation¶

After introducing the abovementioned models, some basic model evaluation techniques are presented.

Logistic Regression¶

A regression model that is $\in[0,1]$ bound, essentially modeling probabilities. Thereafter some cutoff value can be used to split the predicted value $(\hat{y})$ into successes $1$ & failures $0$.

This is done by modeling the $\ln(odds)$, where $odds = \frac{p}{1-p}$ where $p$ is the probability of success.

Fitting the logs odds is analogous to applying an activation function to ensure:

$$\hat{Y} \in [0,1]$$

Additional Notes¶

Logistic Regression is a linear classifier (separates the data by a straight line).

Features are scaled to avoid overweighing variables.

KNN: K Nearest Neighbors¶

Algorithm¶

- Initialize K, the number of neighbors in the dataset (commonly K=5)
- Take the K nearest neighbors (normally Euclidean distance but may be other) of a new datapoint and assign the new datapoint to the the category wherein the majority of its neighbors reside

(remember we are working with labelled data)

Support Vector Machine (SVM)¶

In the case of 2 categories:

Kernal SVM is a linear classifier that finds a decision boundary hyperplane that maximises the distance between the support vectors (2 selected datapoints, one from each category). Referred to as vectors as apposed to points in the general case of multidimentional data.

The support vectors (the 2 selected points, one from each category) are chosen as the nearest two points that decide each category, as such where most classification algorithms learn to distinguish the most obvious cases, SVM instead linearly separates items with far less nuance.

Kernal Support Vector Machine (SVM)¶

A major drawback of a traditional SVM is its linearity. Data is often more accurately separated through non-linear devision. Imagine, for example, a dataset wherein one category is completely surrounded by another, clearly linear separation is inappropriate.

Higher Dimensional Linear Devision¶

Mathematically, it is always possible to transform a dataset to a higher dimensional space & then separate categories linearly. Continuing with the abovemetioned example, the dataset could be converted to a higher diminensional mapping, thereafter the inner category would rise much higher than the outter category and is readily separated with a linear hyperplane.

This higher dimensional process is often infeasible as it is too computationally expensive. The kernal trick offers an inexpensive equivolent.

The Kernal Trick¶

The data is converted to a Guassian Kernal (spherical Guassian) - based on the distance of each datapoint from an optimally chosen midpoint. Thereafter selecting the correct $\sigma$ in $\sim \mathcal{N}(\mu, \sigma)$ accounts for the spread of the distribution. More complecated datasets can be modelled leveraging multiple Gaussian Kernals to capture asymmetrical, non-linear datasets.

The key takeaway is that a non-linear, optimal devision is found without extrapolating the data into a higher dimensional space.

We are not limited to a Gaussian kernal. Other popular choices include the Sigmoid kernal & Polynomial Kernal - both of which essentially capture the distance from the centroid.

Futher reading on Kernal mechanics: https://pdfs.semanticscholar.org/f8a9/51ce919dca7dea476b3a508b2c322a2814f9.pdf

Naive Bayes¶

A Bayesian, probabilistic, approach to classification.

Suppose there are two categories & we are given a new datapoint:

Step 1¶

Calculate the probability that a new datapoint is an example of category 1 given its features:

$$P(\hat{Y}=1|X) = \frac{P(X|\hat{Y}=1)*P(\hat{Y}=1)}{P(X)}$$

Step 2¶

Calculate the probability that a new datapoint is an example of category 2 given its features:

$$P(\hat{Y}=2|X) = \frac{P(X|\hat{Y}=2)*P(\hat{Y}=2)}{P(X)}$$

Diagnostics:¶

Prior: unconditional probability of $\hat{Y}$

Marginal Likelihood:

- place a radius arround the new datapoint
- consider all the existing datapoints inside the radius as 'similar' to the new datapoint
- therefor, P(x) can be calulated as the probability of a datapoint being similar to those in the radius, thus is calculated by P(x) = (no. of points in radius / total obervations)

Likelihood:

- again consider all points in the radius
- calculate the likelihood that the datapoint has features X, given it is of category K.
- P(X|y) = (no. of points in radius in category K / no. of points in radius)

Step 3¶

Once both probabilities are calculated, the algorithm classifies the new point as whichever category offers the highest probability. Clearly if there are two probabilities, calculating only 1 will suffice. Similarly, if there are $K$ categories only $K-1$ need be calculated.

Additional Notes:¶

Why is it called Naive Bayes¶

Bayes theorem relies on some relies on the Independence assumption, it is often Naive to do so.

Computational Efficiency¶

Since we are comparing two Bayesian probabilities, it is sufficent to simply compute the numerators, as denominators cancle out - saving computational effort.

Decision Tree Classifier¶

Perform a series of linear separations to split the data. The splits are done as to maximize the number in each category after the split - a split attempts to minimize infomational entropy.

Notes:¶

If too expensive to search the entire tree, a probabilistic approach can be used to classify datapoints at higher (not at the terminal) nodes - wherein the probability of each category is calculated at a higher node.

Caveat: setting too many decision boundaries will result in overfitting.

Decision trees lost favour in the ML community, but has seen a resergence as slightly more elaborate variation of the algorithm (like Random Forest & Gradient Boosting) has proven very useful.

Random Forest¶

Ensamble Learning¶

Ensample learning is the practice of combining multiple learning algorithms to form a new, more comprehensive, algorithm.

Random Forest Algorithm¶

- Pick a random subset of size K from the data
- Build a decision tree associated with the K points
- Choose n: the number of trees to compute
- Repeat steps 1 & 2 n times
- Prediction of a new datapoint is then done by making each tree PREDICT the category, & thereafter assign the datapoint to the category that wins the majority vote.

The power of number can eradicate erroneous results and improve performance - despite at the cost of computation.

Model Evaluation¶

Type 1 & Type 2 Errors¶

- True Positive
- True Negative
- False Positive (Type 1 Error)
- False Negative (Type 2 Error)

False Postive: Type 1 Error:¶

Predicted True - that did not occur

False Negative: Type 2 Error:¶

Predicted False - that did occur

Confusion Matrix¶

Gives the frequency of actual vs predicted values, indicating: TP, TN, FP, FN. Sklearn confusion matrix is given as: (with reference to table) Top index: $\hat{Y}$: Predicted values Left index: $Y$: Actual values

Thus indicating:

TN | FP

FN | TP

Accuracy:¶

Accuracy Paradox: The accuracy rate can be misleading, as in the case of extreme value data.

$Total = TN + FN + TP + FP$

Accuracy Rate: $$ \frac{TP + TN}{total}$$

Error Rate: $$ \frac{FP + FN}{total}$$

Sensitivity & Specificity¶

(in terms of actual - not predicted - to form the base)

Sensitivity: $$\frac{TP}{TP + FN}$$

Specificity: $$\frac{TN}{TN + FP}$$
